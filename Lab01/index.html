    ¿Cuál es la diferencia entre Internet y la World Wide Web?

    Internet es una red global de redes conectadas entre sí, mientras que la web o la World Wide Web (www) es una recopilación o inmensa colección de información (páginas públicas accesibles) a las que podemos acceder a través de Internet. Asimismo, la World Wide Web es uno de los servicios que se encuentra dentro de Internet.

    ¿Cuáles son las partes de un URL?

    Una url contiene los siguientes elementos:

    • Protocolo HTTPS
    • Subdominio
    • Dominio
    • Extension
    • Sub-carpeta o directorio
    • Parámetro o Query String
    • Etiqueta

    ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    GET: El método de petición GET se encarga de recuperar datos/información de la url a la que se hace el pedido, es decir, obtenemos la información
    HEAD: El método HEAD solicita la misma información que el método GET, sin embargo, este no recibe el cuerpo de la respuesta.
    POST: El propósito del método POST es el envío de información al servidor o crear un nuevo recurso. Asimismo, otro de sus propósitos es manipular datos sensibles.
    PUT: PUT como método HTTP se encarga de reemplazar información que existe.
    PATCH: El método PATCH tiene como propósito aplicar modificaciones parciales a un recurso.
    DELETE: Como su nombre lo menciona, el propósito del método DELETE es borrar información o un recurso existente en el servidor.

    ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método por utilizar sería POST ya que este permite que enviemos información al servidor. Además, los datos que se envían por POST viajan ocultos y queda encriptada la información.

    ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    El método GET se usa cuando accedemos a un sitio por URL o se hace clic en un vínculo.

    Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    El código 200 es una respuesta satisfactoria que significa que la solicitud ha sido exitosa por alguno de los métodos de petición como GET, POST, etc.
    OK → La petición se realizó con éxito.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Depende, porque el error 404 ocurre cuando el servidor no encuentra un recurso que solicito el cliente, es decir, puede tratarse de un contenido que ya no existe o que el desarrollador no quiere que una persona externa acceda. Aunque también puede suceder que el desarrollador haya cometido un error en el enlace y en este caso, el desarrollador si debe verificarlo y/o corregirlo. De igual forma el cliente debe verificar que esta escribiendo correctamente el enlace.
    Not Found → El recurso no fue encontrado.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Sí, porque el error 500 corresponde a un error interno de servidor que por lo general está relacionado con configuraciones de la página.
    Internal Server Error → Hubo un error en el servidor.

    ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Un atributo desaprobado significa que son partes del estándar HTML5 cuyo uso ya no se recomienda. En otras palabras, es un elemento que se considera anticuado ante nuevas estructuras presentadas en actualizaciones.
    Algunos elementos de HTML4 desaprobado en HTML5:

    • applet
    • basefont
    • big
    • center
    • dir
    • font

    ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    Las principales diferencias entre HTML4 y HTML5 es que la última versión de HTML (HTML5) es mucho más dinámica y permite incluir elementos multimedia. Asimismo, ofrece mayor seguridad en el sitio y está menos disponible a exploits debido a que HTML5 es totalmente capaz de realizar cosas sin la necesidad de Javascript o similares. También esta última actualización tiene mejor rendimiento y soporte móvil. Por último, una diferencia importante es que HTML5 se actualiza como un estándar y va cubriendo las necesidades de la web.

    ¿Qué componentes de estructura y estilo tiene una tabla?

    Las tablas en HTML estan conformadas por un título (etiqueta caption) y el cuerpo de la tabla ( etiqueta tbody). Dentro del cuerpo de la tabla se encuentran celdas de encabezado con la etiqueta th, celdas de pie de tabla con la etiqueta tfoot y las celdas normales con la etiqueda de td.
    Cada elemento con td crea una sola celda. A medida que agreguemos más celdas, la fila crecerá.
    Para señalar lo que será una fila, se utiliza la etiqueta tr que significa 'table raw'.
    Para las columnas se tienen dos etiquetas 'col' y 'colgroup'.
    La etiqueta col permite referenciar a una columna, mientras que colgroup permite definir grupos de columnas.

    ¿Cuáles son los principales controles de una forma HTML5?

    Los principales controles de un formulario de HTML5 son: Texto (text, password, email, textarea), verificación (checkbox), opciones (radio), fecha (date), archivos (file) y botones (submit, reset y button)

    Sobre el ciclo de vida y desarrollo de los sistemas de información:

    ¿Cuál es el ciclo de vida de los sistemas de información?

    El ciclo de vida de un Sistema de información consiste en las siguientes fases:

    • Fase de planificación
    • Fase de análisis
    • Fase de diseño
    • Fase de implementación
    • Fase de mantenimiento

    ¿Cuál es el ciclo de desarrollo de sistemas de información?

    El ciclo de desarrollo de un Sistema de información consiste en las siguientes fases:

    • Planificación
    • Definición de requisitos
    • Diseño
    • Desarrollo y pruebas
    • Puesta en marcha
    • Mantenimiento
    • Dispocisión
