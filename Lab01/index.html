Lab01: Introducción a las aplicaciones web, HTML5 y ciclo de vida de los sistspanas de información

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de redes conectadas entre sí, mientras que la web o la World Wide Web (www) es una recopilación o inmensa colección de información (páginas públicas accesibles) a las que podspanos acceder a través de Internet. Asimismo, la World Wide Web es uno de los servicios que se encuentra dentro de Internet.

¿Cuáles son las partes de un URL?

Una url contiene los siguientes elspanentos:

  • Protocolo HTTPS
  • Subdominio
  • Dominio
  • Extension
  • Sub-carpeta o directorio
  • Parámetro o Query String
  • Etiqueta

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: El método de petición GET se encarga de recuperar datos/información de la url a la que se hace el pedido, es decir, obtenspanos la información
HEAD: El método HEAD solicita la misma información que el método GET, sin spanbargo, este no recibe el cuerpo de la respuesta.
POST: El propósito del método POST es el envío de información al servidor o crear un nuevo recurso. Asimismo, otro de sus propósitos es manipular datos sensibles.
PUT: PUT como método HTTP se encarga de respanplazar información que existe.
PATCH: El método PATCH tiene como propósito aplicar modificaciones parciales a un recurso.
DELETE: Como su nombre lo menciona, el propósito del método DELETE es borrar información o un recurso existente en el servidor.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejspanplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método por utilizar sería POST ya que este permite que envispanos información al servidor. Adspanás, los datos que se envían por POST viajan ocultos y queda encriptada la información.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método GET se usa cuando accedspanos a un sitio por URL o se hace clic en un vínculo.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 es una respuesta satisfactoria que significa que la solicitud ha sido exitosa por alguno de los métodos de petición como GET, POST, etc.
OK → La petición se realizó con éxito.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende, porque el error 404 ocurre cuando el servidor no encuentra un recurso que solicito el cliente, es decir, puede tratarse de un contenido que ya no existe o que el desarrollador no quiere que una persona externa acceda. Aunque también puede suceder que el desarrollador haya cometido un error en el enlace y en este caso, el desarrollador si debe verificarlo y/o corregirlo. De igual forma el cliente debe verificar que esta escribiendo correctamente el enlace.
Not Found → El recurso no fue encontrado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, porque el error 500 corresponde a un error interno de servidor que por lo general está relacionado con configuraciones de la página.
Internal Server Error → Hubo un error en el servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elspanentos de HTML 4 que en HTML5 estén desaprobados.

Un atributo desaprobado significa que son partes del estándar HTML5 cuyo uso ya no se recomienda. En otras palabras, es un elspanento que se considera anticuado ante nuevas estructuras presentadas en actualizaciones.
Algunos elspanentos de HTML4 desaprobado en HTML5:

  • applet
  • basefont
  • big
  • center
  • dir
  • font

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Las principales diferencias entre HTML4 y HTML5 es que la última versión de HTML (HTML5) es mucho más dinámica y permite incluir elspanentos multimedia. Asimismo, ofrece mayor seguridad en el sitio y está menos disponible a exploits debido a que HTML5 es totalmente capaz de realizar cosas sin la necesidad de Javascript o similares. También esta última actualización tiene mejor rendimiento y soporte móvil. Por último, una diferencia importante es que HTML5 se actualiza como un estándar y va cubriendo las necesidades de la web.

¿Qué componentes de estructura y estilo tiene una tabla?

Las tablas en HTML estan conformadas por un título (etiqueta caption) y el cuerpo de la tabla ( etiqueta tbody). Dentro del cuerpo de la tabla se encuentran celdas de encabezado con la etiqueta th, celdas de pie de tabla con la etiqueta tfoot y las celdas normales con la etiqueda de td.
Cada elspanento con td crea una sola celda. A medida que agreguspanos más celdas, la fila crecerá.
Para señalar lo que será una fila, se utiliza la etiqueta tr que significa 'table raw'.
Para las columnas se tienen dos etiquetas 'col' y 'colgroup'.
La etiqueta col permite referenciar a una columna, mientras que colgroup permite definir grupos de columnas.

Lab02: CSS

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

!important es una declaración muy poco usada que permite declarar un atributo de forma que este tenga prevalencia sobre otras declaraciones posteriores, en otras palabras, desativa las reglas de cascada.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Una imagen de fondo debe escogerse con cuidado debido a distintos motivos, por ejemplo: las distintas definiciones de pantalla, el color de texto del sitio y los pixeles de la imagen.
Si utilizamos una imagen con pocos pixeles, el ordenador deberá sobretrabajar para crear el mosaico repitiendo la imagen, la cantidad de veces sea necesaria.
Algo similar sucede cuando se ve la imagen de fondo en distintas definiciones de pantalla. En una pantalla puede verse bien, sin embargo, en otras puede reducirse o bien se ve un mosaico del fondo.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Dependiendo de la necesidad del diseño del sitio web. No obstante, si se busca que el sitio sea responsivo es recomendable que se utilicen unidades relativas como em, % o rem.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

La versión minimizada del CSS mejora el rendimiento debido a que se eliminan datos innecesarios o redundantes de un recurso sin que se vea afectada la forma en que los navegadores lo procesan.

Lab05: Fundamentos de JavaScript

¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Java es un lenguaje que hay que compilar y para ello es necesario utilizar la JVM (Java Virtual Machine) para que el código pueda ser interpretado por la «máquina».
JavaScript es un lenguaje interpretado ya que se lee y traduce directamente a medida que se ejecuta el programa. Además, en sus inicios, este lenguaje fue pensado para utilizarse principalmente en navegadores web como Firefox, Chrome o Safari;
Algunas diferencia entre estos dos lenguajes son:

  • Compilador: Java necesita un Kit de desarrollo y un compilador. Mientras que, Javascript no necesita que sus programas se compilen
  • Orientado a objetos: Java es un lenguaje orientado a objetos. Javascript es un lenguaje "multiparadigma" no requiere programar orientado a objetos, sin embargo, sí lo permite.
  • Propósito: Java es un lenguaje de propósito general y Javascript sólo podemos escribir programas para que se ejecuten en páginas web o para aplicaciones como programas de consola, aplicaciones para móviles, de escritorio, etc.
  • Tipado estático: Java es un lenguaje de programación fuertemente tipado. Mientras que Javascript es un lenguaje de tipado dinámico.
  • Lenguaje abierto / lenguaje propietario: Javascript está basado en un estándar abierto, es decir, cualquier fabricante puede implementarlo en sus sistemas libremente. Sin embargo, Java es un lenguaje propiedad de una empresa, por lo que está dirigido con un enfoque particular y comercial.
Algunas semejanzas entre estos dos lenguajes son:
  • Clases: Ambos lenguajes de programación contienen clases
  • Orientado a Objetos: Tanto Java y JavaScript puede utilzarse como orientado a objetos
