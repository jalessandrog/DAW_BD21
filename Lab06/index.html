Lab06: Programación orientada a eventos

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Gracias a que se puede validar los inputs desde Javascript se evita que se envíe y guarde información erronea en el servidor y de esta forma evitar problemas futuros en la base de datos, o en general, en todo aquello correspondiente al backend.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Existen diversas formas para saltarse la seguridad, dos de ellas son:

  • Desactivar del navegador el lenguaje de JavaScript
  • Modificando desde la consola las input que tienen algun requisito obligatorio

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque con JavaScript solo se realizarán validaciones para evitar enviar al servidor datos con errores, eso no sustituye las practicas de seguridad que el desarrollador debe implementar en la aplicación web. Por ejemplo, librerias de NodeJS como Bcrypt para cifrar contraseñas, etc.

Ejercicio 1

Desarrolla una página para validar passwords. La página debe tener una forma con 2 campos, el campo de password, y el campo de verificar password. Utiliza al máximo tu creatividad e ingeniería para que la página sea un validador de passwords de estado del arte, con la mejor experiencia para el usuario.
